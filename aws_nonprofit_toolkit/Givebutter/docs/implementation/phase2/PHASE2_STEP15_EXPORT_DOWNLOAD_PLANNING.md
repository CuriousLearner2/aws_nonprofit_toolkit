## Executive Summary

Plan how reviewers can download previously generated export files safely, without arbitrary file access, source-data mutation, CRM writeback, or unauthorized filesystem exposure. The download mechanism must be auditable, tied to the generating import batch, and use the audit log as the source of truth for file metadata.

**Core Principle:** Download streaming exposes only export files generated by the application for the current import batch. Never allow arbitrary file paths or user-controlled filesystem access.

## Part 2: Proposed Download Scope

### First Scope Recommendation: Audit-Log-Based Download

**Principle:**
- Use `audit_log_id` to locate files
- Verify audit record belongs to import batch
- Verify action_type is "export_generated"
- Read filename/path from audit details
- Validate path safety
- Stream file as CSV attachment

**Why Audit Log Approach:**
1. ✅ Audit record already tied to batch_id
2. ✅ Filename and path stored in audit details
3. ✅ Reduces user-controlled input risk
4. ✅ Allows future audit trail for downloads (if needed)
5. ✅ Separates file ownership verification from file delivery

**What NOT to Accept:**
- ❌ Direct filename from request (path traversal risk)
- ❌ Absolute paths from request
- ❌ Directory listing/browsing
- ❌ Wildcard patterns
- ❌ Symlinks outside export directory
- ❌ Files not referenced in audit log

### Scope Boundaries

**In Scope:**
- ✅ Download CSV files generated by `export_generated` action
- ✅ Files for current import batch only
- ✅ Path validation against configured export directory
- ✅ Standard HTTP attachment download

**Out of Scope (defer to later):**
- ❌ CRM/Givebutter writeback
- ❌ Remote upload
- ❌ File regeneration during download
- ❌ Export history table (audit log only)
- ❌ Auth/login implementation
- ❌ Role-based access control
- ❌ Signed URLs
- ❌ Retention cleanup jobs
- ❌ Multi-format downloads
- ❌ Download audit trail (optional, decide in implementation)

## Part 3: Route/API Shape

### Proposed Route

**Endpoint:** `GET /imports/<import_id>/exports/download/<audit_log_id>`

**Rationale:**
- audit_log_id is stable and unique
- Ties request to specific generation event
- Avoids user-controlled filenames in URL
- Allows server to verify batch ownership

**Alternative Considered:** `GET /imports/<import_id>/exports/file/<filename>`
- **Pro:** More RESTful filename in URL
- **Con:** Requires filename validation; filename user-controlled in URL; higher path traversal risk
- **Decision:** Prefer audit_log_id approach

### Route Behavior

**Request:**
```
GET /imports/IMP-2025-0101-A/exports/download/12345
```

**Response on Success (200):**
```
Content-Type: text/csv
Content-Disposition: attachment; filename="IMP-2025-0101-A_export_20260612_143022.csv"
Content-Length: 45678

<CSV content stream>
```

**Response on Missing Audit Record (404):**
```json
{
  "error": "Export file not found",
  "message": "No export record found for this import"
}
```

**Response on Wrong Batch (403 or 400):**
```json
{
  "error": "Access denied",
  "message": "This export does not belong to the specified import batch"
}
```

**Response on File Not Found (404):**
```json
{
  "error": "File not found",
  "message": "The export file no longer exists on disk"
}
```

**Response on Path Safety Error (500):**
```json
{
  "error": "Internal error",
  "message": "Export file path validation failed"
}
```

### Handler Pseudocode

```python
@app.route('/imports/<import_id>/exports/download/<int:audit_log_id>', methods=['GET'])
def download_export(import_id, audit_log_id):
    """Download previously generated export file."""
    try:
        # Get audit record
        audit_record = db.query(AuditLogRecord).filter_by(
            id=audit_log_id,
            batch_id=import_id,
            action_type="export_generated"
        ).first()
        
        if not audit_record:
            return jsonify({"error": "Export not found"}), 404
        
        # Extract filename/path from audit details
        details = audit_record.details
        filename = details.get('filename')
        file_path = details.get('file_path')
        
        # Validate path safety
        download_info = get_export_download_info(
            import_id=import_id,
            audit_log_id=audit_log_id,
            export_dir=current_app.config['EXPORT_OUTPUT_DIR'],
        )
        
        # Stream file
        return send_file(
            download_info.file_path,
            as_attachment=True,
            download_name=download_info.filename,
            mimetype=download_info.content_type,
        )
        
    except ValueError as e:
        return jsonify({"error": "Invalid request"}), 400
    except FileNotFoundError as e:
        return jsonify({"error": "File not found"}), 404
    except Exception as e:
        logger.error(f"Download error: {e}")
        return jsonify({"error": "Download failed"}), 500
```

## Part 4: Path Safety Policy

### Validation Rules

**1. Audit Record Validation**
- Record must exist with matching `id` and `batch_id`
- Record must have `action_type = "export_generated"`
- Record must have non-empty `details.file_path`

**2. Path Resolution**
- Use `Path.resolve()` to resolve symlinks and `..` sequences
- Resolve both file path and export directory to absolute paths
- Compare resolved paths

**3. Directory Containment**
- Verify resolved file path is within resolved export directory
- Use `.is_relative_to()` (Python 3.9+) or equivalent check
- Reject if file is outside directory (symlink escape attempt)

**4. File Existence**
- Verify file exists on disk before streaming
- Return 404 if file missing (e.g., manual deletion)

**5. Rejection Rules**
- Reject if path contains `..` segments
- Reject if path is absolute from request
- Reject if resolved path escapes export directory
- Reject if file doesn't exist
- Reject if audit details has no path

### Validation Pseudocode

```python
def validate_export_path(file_path: str, export_dir: str) -> bool:
    """Validate that file_path is safe and within export_dir."""
    export_path = Path(export_dir).resolve()
    file_path_resolved = Path(file_path).resolve()
    
    # Check if file is within export directory
    try:
        file_path_resolved.relative_to(export_path)
    except ValueError:
        # file_path is outside export_dir
        return False
    
    # Check file exists
    if not file_path_resolved.exists():
        return False
    
    # Check it's a file, not directory
    if not file_path_resolved.is_file():
        return False
    
    return True
```

### Service Function

**Function:** `get_export_download_info()`

```python
def get_export_download_info(
    import_id: str,
    audit_log_id: int,
    export_dir: str,
    config: Optional[Mapping[str, Any]] = None,
) -> ExportDownloadInfo:
    """
    Get validated export download information.
    
    Verifies:
    - Audit record exists and belongs to import_id
    - action_type is "export_generated"
    - File path is safe and within export directory
    - File exists on disk
    
    Does NOT:
    - Mutate any data
    - Create audit records
    - Perform I/O beyond path validation
    - Return file content
    
    Args:
        import_id: Import batch ID
        audit_log_id: Audit log record ID
        export_dir: Configured export directory
        config: Optional database configuration
    
    Returns:
        ExportDownloadInfo with validated path and metadata
    
    Raises:
        ValueError: If record not found or invalid
        FileNotFoundError: If file doesn't exist
        RuntimeError: If path validation fails
    """
```

**Implementation Pseudocode:**

```python
def get_export_download_info(
    import_id: str,
    audit_log_id: int,
    export_dir: str,
    config: Optional[Mapping[str, Any]] = None,
) -> ExportDownloadInfo:
    # Get audit record
    repo = get_import_repository(config)
    record = repo.get_audit_record(audit_log_id)
    
    if not record:
        raise ValueError(f"Audit record {audit_log_id} not found")
    
    if record.batch_id != import_id:
        raise ValueError("Audit record does not belong to this import")
    
    if record.action_type != "export_generated":
        raise ValueError("This is not an export record")
    
    # Extract from details
    details = record.details
    filename = details.get('filename')
    file_path = details.get('file_path')
    
    # Validate path safety
    if not _validate_export_path(file_path, export_dir):
        raise RuntimeError("Path validation failed")
    
    # Return validated info
    return ExportDownloadInfo(
        import_id=import_id,
        audit_log_id=audit_log_id,
        filename=filename,
        file_path=file_path,
        content_type="text/csv",
        generated_at=record.action_timestamp,
        row_count=details.get('row_count', 0),
        warning_count=details.get('warning_count', 0),
    )
```

### Path Validation Function

```python
def _validate_export_path(file_path: str, export_dir: str) -> bool:
    """Validate that file_path is safe and within export_dir."""
    try:
        export_path = Path(export_dir).resolve()
        file_path_resolved = Path(file_path).resolve()
        
        # Must be within export directory
        file_path_resolved.relative_to(export_path)
        
        # Must exist and be a file
        if not file_path_resolved.is_file():
            return False
        
        return True
    except (ValueError, OSError):
        return False
```
